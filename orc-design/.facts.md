# Apache ORC 源码核实事实台账（source-verified fact ledger）

> 源码：`~/workdir/orc`，主锚 C++ 实现（`c++/src/`、头 `c++/include/orc/`），规范 `site/specification/ORCv1.md`。每条事实从真实源码 grep/read 核实,标 `file:line`(相对仓库根)。
> 判型:**自描述列式文件格式 + 读写库**(self-describing columnar file format + R/W library) —— 不是存储引擎/表服务;定义**单文件**物理布局(header+stripes+footer+postscript),嵌入宿主(Hive/Spark/Iceberg)使用。
> 边界:ORC 只管**一个文件**。多文件表、跨文件 schema 演进、ACID 由外部引擎管;ORC 内只做单文件自描述 + 读侧类型转换(SchemaEvolution)。
> 设计张力:**列式压缩率/扫描剪枝(按列投影 + 三级统计/SARG 跳 stripe+row-group,只读选中列)vs 写复杂度(必须缓冲整个 stripe、先建 index+stats 才能 flush)**;并 **自描述单文件(footer+postscript 自定位,无需外部元数据即可读)vs 多文件表/ACID 交给引擎**;并 **index-then-data 物理布局 vs seek 粒度(row-group=10000 行)**。

## 判型依据(单文件自描述 + 库边界)
- 文件三段:Header("ORC" 3 字节) + Body(stripes:index/data/footer) + Tail(metadata/file footer/postscript/psLen) —— `site/specification/ORCv1.md:32-53`、`:107-113`。
- 写侧 `c++/src/Writer.cc:363 magicId="ORC"`;读侧库入口 `c++/src/Reader.cc:1631 createReader`。
- 库对外:`c++/include/orc/Writer.hh`(WriterOptions/createWriter)、`Reader.hh`(ReaderOptions/RowReaderOptions/createReader)、`Type.hh`、`Vector.hh`(ColumnVectorBatch)。宿主提供 InputStream/OutputStream(`c++/src/io/`)。
- 无网络/无多节点/无事务:全部逻辑是"把 batch 编码进一个文件 / 从一个文件解码出 batch"。

## 主线 1 · 文件物理布局与自定位(file layout · postscript 反向解析)
- Tail 三段(+加密统计):encrypted stripe stats / stripe stats(Metadata) / footer(Footer) / postscript(PostScript) / psLen(1 字节) —— `site/specification/ORCv1.md:48-53`。
- **postscript 永不压缩,结束于文件倒数第 1 字节;最后 1 字节 = postscript 长度** —— `site/specification/ORCv1.md:60-73`;读侧 `c++/src/Reader.cc:1664 postscriptLength = buffer->data()[readSize-1] & 0xff`(<256)。
- 反向读:先读尾部 `min(16KB, fileLength)` —— `c++/src/Reader.cc:1657 readSize=min(fileLength, DIRECTORY_SIZE_GUESS)`;`c++/src/Reader.hh:36 DIRECTORY_SIZE_GUESS = 16*1024`。一次读期望同时拿到 footer+postscript。
- 校验 magic:`c++/src/Reader.cc:1514 ensureOrcFooter`,`:1515 MAGIC("ORC")`,末尾或开头比对。
- 解析链:`readPostscript`(`:1545`)→ 由 ps 得 footer_length → `readFooter`(`:1612`,按 ps 压缩类型解压 protobuf)。
- FileTail 可序列化缓存复用(免二次读尾):`c++/src/Reader.cc:1641-1651 getSerializedFileTail` 分支。
- 不变量:**读文件只需文件自身**(尾部自带类型/统计/stripe 目录),无外部 catalog。

## 主线 6 · 行组与行索引(row groups · row index · seek 粒度)
- 默认 row-group = **10000 行**:`c++/src/Writer.cc:56 rowIndexStride=10000`;`site/specification/ORCv1.md:1244`。
- 写:`c++/src/ColumnWriter.cc:174 createRowIndexEntry()` → `:188 recordPosition()`;`:238 recordPosition` 让各 encoder 记录流内位置。
- **RowIndexEntry = positions[] + statistics** —— `site/specification/ORCv1.md:1254-1257`。
- positions 语义:未压缩流 = RLE run 起始字节偏移 + run 内已消费值数;压缩流 = 压缩块起点 + 解压后字节数 + RLE 已消费值数 —— `site/specification/ORCv1.md:1266-1272`。多流位置串接、顺序固定(`:1274-1286`)。
- **index 流置于 stripe 最前**:默认流式读不需读它,仅 PPD 或 seek 时加载 —— `site/specification/ORCv1.md:1248-1251`。

## 五路径归纳(供绘图 · 总架构图重画依据)
- **Hot Path**:
  - 写:`add(batch)` → 按列 ColumnWriter 树编码 → 满 stride 建 row-index → 满 stripeSize `writeStripe`(字典→index→data→stripeFooter) → close 写 metadata/footer/postscript。
  - 读:`createReader`(读尾 16KB→postscript→footer) → `RowReaderImpl::next` → startNextStripe(剪枝) → 按选中列 ColumnReader 树解码 → ColumnVectorBatch。
- **Control Path**:SARG 三级剪枝(file stats → stripe stats →〔字典〕→ row-group pickRowGroups),TruthValue 7 值格 + isNeeded;列投影 selectedColumns;seekToRow/seekToRowGroup(positions)。
- **State Path**:单文件自描述状态 —— Type 树(列 ID 前序)/ 三级统计(rg/stripe/file)/ row-index(positions+stats)/ 每列多流(PRESENT/DATA/LENGTH/DICT/SECONDARY);postscript 自定位。
- **Failure Path**:ensureOrcFooter magic 校验、tail size / StripeInformation 越界校验(`Reader.cc:1670`、`:1258-1267`)、老 writer(<ORC-135)时间戳禁 PPD、缺 hasNull 退化 YES_NO_NULL;跨文件/事务失败不归 ORC(交引擎)。
- **Optimization**:列投影只读选中流、三级统计+SARG 跳 stripe/row-group、bloom filter、RLE v1/v2(SHORT_REPEAT/DIRECT/PATCHED_BASE/DELTA)、字典编码(按基数比自适应回退)、per-stream 压缩(ZSTD 默认)、index-then-data 布局(流式读免读索引)、FileTail 序列化缓存。

## 总架构图重画蓝图(单文件工业剖面)
横向文件剖面(左→右按物理偏移):
- **左:Header** "ORC"(3 字节)。
- **中:Body = N × Stripe**,每个 stripe 竖切三层:① Index streams(ROW_INDEX + BLOOM_FILTER,含 positions + row-group stats,置最前)→ ② Data streams(按列:PRESENT/DATA/LENGTH/DICTIONARY_DATA/SECONDARY)→ ③ Stripe Footer(streams 目录 + column encodings,唯一定位真相)。stripe 只含整行、行不跨界。
- **右:Tail**(反向自定位):Metadata(stripe 级统计) → File Footer(Type 树 + stripe 目录 + file 级统计 + rowIndexStride) → PostScript(压缩类型/footer 长度,不压缩) → **最后 1 字节 = psLen**。
挂两条竖轴贯穿全图:
- **写轴(下→上聚合)**:batch → ColumnWriter 树 → row-index/stats/字典 → flush stripe → footer/postscript。
- **读轴(上→下下推)**:postscript→footer→SARG 三级剪枝(file→stripe→row-group)+ 列投影 → 只读选中列的选中 row-group → ColumnVectorBatch。
- **贯穿主键**:列 ID(前序,根=0)串起 types / streams / statistics / row-index / SARG。
- **边界虚线**:单文件之外(多文件表 / schema 跨文件演进 / ACID)标注"交 Hive/Iceberg/Delta 引擎",ORC 内只留 SchemaEvolution 读侧类型适配。
