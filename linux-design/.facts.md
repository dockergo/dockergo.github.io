# Linux 内核 源码核实事实台账（source-verified fact ledger）

> 源码：`~/workdir/linux-7.1.3`（VERSION 7.1.3）。每条事实从真实源码 grep/read 核实，标 `file:line`。
> 判型:**宏内核操作系统**(monolithic OS kernel) —— 所有子系统共处一个地址空间、彼此直接函数调用；系统调用为唯一接触面。它是"母隐喻"本身：所有其它项目运行其上的那台计算机器。
> 四层剖面:接触面(syscall) → 核心子系统(sched/mm/vfs/net/block) → 支撑机制(irq/lock/timer/device/isolation) → 硬件。
> 设计张力:**宏内核性能(同址直调、快)vs 复杂度/隔离性(任一处 bug 可 panic 全核)**;并含 **抢占低延迟 vs 吞吐**、**通用抽象(sched_class/VFS/device model 统一接口)vs 特化性能路径**。

## 五路径归纳（供绘图 · 总架构图重画依据）
- **Hot Path**:用户 syscall → `do_syscall_64`(表分派)→ 命中子系统(read/write→VFS→page cache→blk-mq;send→socket→协议栈→NIC;缺页→handle_mm_fault);CPU 竞争经 `__schedule`→`pick_next_task_fair`(EEVDF)切换。
- **Control Path**:fork/exec/exit 生命周期(kernel_clone/copy_process);调度类多态(sched_class/for_each_class);cgroup 配额 + namespace 视图;device match→probe 绑定。
- **State Path**:task_struct(PCB)+ mm_struct/VMA(地址空间)+ file/inode/dentry(VFS)+ sk_buff(报文)+ page cache;RCU/rbtree/wheel 承载的内核对象图。
- **Failure Path**:缺页/异常 → do_user_addr_fault;任一处 bug 无隔离 → panic 全核(宏内核代价);锁死/优先级反转靠 mutex handoff、RCU。
- **Optimization**:NAPI 批收(软中断)、blk-mq 多队列、per-VMA RCU 锁、qspinlock MCS 队列、RCU 读无锁、fair 类直取快路径、hrtimer 红黑树、页缓存。

## 总架构图重画蓝图（四层内核工业剖面）
自顶向下四带,横向按子系统并列:
- **顶层 接触面(User/Kernel 栅栏)**:用户进程 → `entry_SYSCALL_64` → `do_syscall_64` → sys_call_table 分派。唯一入口,一进一出各一次态切换。
- **核心子系统带(并列五柱,同址直调互连)**:
  - 进程/调度:task_struct → `__schedule` → `pick_next_task_fair`(EEVDF)→ context_switch;per-CPU rq。
  - 虚拟内存:mm_struct/VMA → `handle_mm_fault` → 页表。
  - VFS:file/inode/dentry → `vfs_read/write` → page cache。
  - 网络:socket → 协议栈 → `net_rx_action`/NAPI → sk_buff。
  - 块IO:`submit_bio` → blk-mq → request。
- **支撑机制带(横贯托底)**:中断/软中断(__do_softirq 向量)、同步(spinlock/mutex/rwsem/RCU)、时间(jiffies/tick/hrtimer/timer wheel)、设备模型(device/driver/bus probe)、隔离(cgroup 配额 + namespace 视图)。
- **底层 硬件**:CPU/MMU/NIC/磁盘/时钟;向上以中断进、以驱动 probe 绑定。
- **贯穿注记**:全部处于单一内核地址空间、直接函数调用(宏内核性能);代价是无进程级隔离——任一子系统 bug 可 panic 全核(设计张力核心)。抢占点(PREEMPT 模型)、通用抽象(sched_class/VFS/device model)与特化快路径(NAPI/blk-mq/per-VMA 锁/RCU)贯穿其间。
