## 判型（一句定性）

**所有权驱动的系统级语言 —— 编译期借用检查（无 GC 内存安全）+ 单态化零成本抽象 + MIR 借用检查器为灵魂。**

支撑：
- 借用检查在 **MIR** 上进行，crate 头注释自证：`compiler/rustc_borrowck/src/lib.rs:1` = `//! This crate implements MIR typeck and MIR borrowck.`
- 无 GC 内存安全靠 RAII/Drop：`library/core/src/ops/drop.rs:209 pub const trait Drop`，析构由 `library/core/src/ptr/mod.rs:721 drop_in_place` 执行。
- 单态化零成本抽象：`compiler/rustc_monomorphize/src/collector.rs:1471 fn collect_roots` / `:1821 collect_crate_mono_items`。
- Send/Sync 编译期并发安全：`library/core/src/marker.rs:92 pub unsafe auto trait Send` / `:657 pub unsafe auto trait Sync`。

## 设计张力（显式对置）

**编译期严格**（借用检查 / 生命周期 / 单态化 → 学习陡峭 + 编译较慢）
　⟷　
**运行期零成本**（无 GC、无运行时、内存安全 + 无畏并发）

- 严格侧：`mir_borrowck` 过不了就不编译（`compiler/rustc_borrowck/src/lib.rs:115 fn mir_borrowck`）；单态化按具体类型展开导致编译体量/时间上升（`collector.rs` + `partitioning.rs:143 fn partition`）。
- 零成本侧：运行期无 GC / 无运行时类型（`library/` 无 runtime GC 机制），仅剩 Drop 析构、Arc 计数、panic 展开。

## 五路径归纳

### ① Hot · 编译主链（绿，compile-flow）
源码经固定 query 链一路降级到机器码。驱动：`compiler/rustc_interface/src/interface.rs:384 fn run_compiler` → `passes.rs:927 create_and_enter_global_ctxt` → `passes.rs:1186 fn analysis`。
1. lex/parse → AST：`compiler/rustc_parse/src/lib.rs:93 new_parser_from_source_str` / `:108 new_parser_from_file`
2. AST → HIR：`compiler/rustc_ast_lowering/src/lib.rs:666 fn lower_to_hir`（lib.rs:1 `//! Lowers the AST to the HIR.`）
3. 类型/trait 检查（HIR/THIR）：`compiler/rustc_hir_typeck/src/lib.rs:87 fn typeck_root`；总驱动 `passes.rs:1135 rustc_hir_analysis::check_crate`
4. THIR → MIR 构建：`compiler/rustc_mir_build/src/builder/mod.rs:67 build_mir_inner_impl` / `:451 construct_fn`；THIR 模块 `compiler/rustc_mir_build/src/thir/`
5. 借用检查（MIR 上）：`compiler/rustc_borrowck/src/lib.rs:115 fn mir_borrowck`
6. 单态化 + MIR 优化：`compiler/rustc_monomorphize/src/collector.rs:1471 collect_roots`；`compiler/rustc_mir_transform/src/inline.rs`、`dataflow_const_prop.rs`
7. codegen → LLVM → 机器码：`compiler/rustc_codegen_ssa/src/base.rs:691 codegen_crate` / `:395 codegen_instance`；后端 `compiler/rustc_codegen_llvm/src/base.rs:65 compile_codegen_unit`

### ② Control · 借用检查裁决（蓝，borrow-check）
借用检查器横切 MIR，是"灵魂"。裁决三件事：所有权、借用别名、生命周期（region）。
- 入口：`compiler/rustc_borrowck/src/lib.rs:115 fn mir_borrowck`
- NLL（非词法生命周期）：`compiler/rustc_borrowck/src/nll.rs`
- Polonius（下一代 region 推断）：`compiler/rustc_borrowck/src/polonius/`；`nll.rs:8 use polonius_engine::{Algorithm, AllFacts, Output}`
- region 推断（生命周期求解）：`compiler/rustc_borrowck/src/region_infer/mod.rs`、`universal_regions.rs`
- unsafe 检查（MIR build 期）：`compiler/rustc_mir_build/src/check_unsafety.rs:1055 fn check_unsafety`

### ③ State · 所有权与生命周期（琥珀，ownership）
所有权模型的运行期落地 = 智能指针族 + Drop + 内部可变。
- Box（独占堆）：`library/alloc/src/boxed.rs:236 pub struct Box`
- Rc（单线程引用计数）：`library/alloc/src/rc.rs:320 pub struct Rc`
- Arc（原子引用计数，跨线程）：`library/alloc/src/sync.rs:269 pub struct Arc`
- Cell / RefCell / UnsafeCell（内部可变）：`library/core/src/cell.rs:312 Cell` / `:849 RefCell` / `:2323 UnsafeCell`
- Drop（RAII 出作用域析构）：`library/core/src/ops/drop.rs:209 pub const trait Drop`；`library/core/src/ptr/mod.rs:721 drop_in_place`

### ④ 失败 · borrow-error（橙，error path）
过不了借用检查 → 直接编译报错，不进入 codegen。
- move 后使用 / 借用冲突诊断：`compiler/rustc_borrowck/src/diagnostics/move_errors.rs:106 report_move_errors`、`conflict_errors.rs`
- region/生命周期错误：`compiler/rustc_borrowck/src/diagnostics/region_errors.rs`、`explain_borrow.rs`

### ⑤ 横切 · trait 单态化（紫，trait-mono）
trait 求解 + 泛型单态化 = 零成本抽象；默认静态分发，`dyn` 才虚表。
- trait 选择/求解：`compiler/rustc_trait_selection/src/traits/select/mod.rs:289 fn select`；下一代求解器 `compiler/rustc_trait_selection/src/solve/`
- 类型合一（union-find 推断）：`compiler/rustc_infer/src/infer/`（`unify_key.rs`、`region_constraints/`、`lexical_region_resolve/`）
- 单态化收集：`compiler/rustc_monomorphize/src/collector.rs:1471 collect_roots` / `:1821 collect_crate_mono_items`
- codegen 单元划分：`compiler/rustc_monomorphize/src/partitioning.rs:143 fn partition` / `:1144 collect_and_partition_mono_items`

### 运行期薄层 · panic 与展开（错误路径终点）
- unwind（逐层析构可捕获）：`library/panic_unwind/`；`library/std/src/panic.rs:358 fn catch_unwind`
- abort（直接终止不析构）：`library/panic_abort/`
