## 判型（一句锐利定系统类型）

**基于 RDD 血缘（lineage）的分布式内存计算引擎**：一个 Driver 把惰性算子汇成 DAG，
`DAGScheduler` 按 **shuffle 边界**切 Stage、按分区拆 Task，`TaskScheduler` 分派到 N 个同构
Executor 并行跑；**惰性求值 + 血缘容错重算**——丢分区靠血缘重算，不靠全量副本。数据在外部存储，
Spark 不自管持久存储。

支撑锚点：
- `core/src/main/scala/org/apache/spark/scheduler/DAGScheduler.scala:66` 注释原文
  "Spark stages are created by breaking the RDD graph at shuffle boundaries"。
- `core/src/main/scala/org/apache/spark/rdd/RDD.scala:84` `abstract class RDD`；`:116` `def compute`；
  `:260` `final def dependencies`；`:334` `final def iterator`（惰性求值入口）。
- `core/src/main/scala/org/apache/spark/SparkContext.scala:603` `_dagScheduler = new DAGScheduler(this)`（每 app 一个 Driver）。

## 设计张力（显式 X vs Y 铭牌）

**惰性血缘换容错（丢分区按 lineage 重算，无需全量副本） vs shuffle 物化 + 内存 spill 的 IO 现实代价。**

- 血缘侧：`RDD.scala:260` `dependencies` 记录父 RDD；`Dependency.scala:52` `NarrowDependency`（窄依赖，父分区局部重算）
  vs `Dependency.scala:84` `ShuffleDependency`（宽依赖，跨 stage 屏障、需重跑上游 map stage）。
  重算入口 `RDD.scala:369` `computeOrReadCheckpoint` / `:381` `getOrCompute`；`:1686` `checkpoint()` 可截断血缘。
- IO 代价侧：shuffle 必须把 map 输出**物化落盘**（`shuffle/sort/SortShuffleWriter.scala:85` `writePartitionedMapOutput`），
  内存不够时**外排 spill**（`util/collection/Spillable.scala:86` `maybeSpill`；`util/collection/ExternalSorter.scala:234` `maybeSpillCollection` / `:262` `spills += spillFile`）。
- 内存侧软边界：`memory/UnifiedMemoryManager.scala:34-56` 注释——执行/存储互借，`spark.memory.fraction=0.6`、
  `spark.memory.storageFraction=0.5`（默认存储区 = 0.6×0.5 = 0.3 堆），执行可回收借出的存储内存（驱逐 cached 块），
  但**存储永不驱逐执行**。

## 五路径归纳（Hot / Control / State / Failure / Optimization）

- **Hot（执行热流 · 绿）**：算子在 Driver 汇成 DAG → Task 在 Executor 线程池跑分区。
  `SparkContext.scala:86` → `DAGScheduler.scala:1058` `submitJob` / `:1117` `runJob` →
  `executor/Executor.scala:551` `launchTask` / `:687` `class TaskRunner`（`:307` `threadPool`）。
- **Control（调度/资源 · 蓝）**：切 stage、分派 task、起 executor、状态回报。
  `DAGScheduler.scala:602` `getOrCreateShuffleMapStage`、`:778` `createResultStage`、`:911` `getMissingParentStages`；
  `scheduler/TaskSchedulerImpl.scala:243` `submitTasks`、`:512` `resourceOffers`、`:798` `statusUpdate`。
  Stage 类型：`Stage.scala:56` / `ShuffleMapStage.scala:37` / `ResultStage.scala:30`。
- **State（血缘 + 存储 · 琥珀）**：RDD lineage、BlockManager 缓存、shuffle 输出位置。
  `storage/BlockManager.scala:182`（`:240` `memoryStore`、`:245` `diskStore`）；
  `storage/BlockManagerMaster.scala:34`（driver 端记录所有 block 位置，`BlockManagerMasterEndpoint.scala:51`）；
  `MapOutputTracker.scala:798` `MapOutputTrackerMaster`、`:932` `registerShuffle`、`:678` `getMapSizesByExecutorId`。
- **Failure（失败重算 · 橙）**：丢 executor / 丢分区 → 按 lineage 重算。
  窄依赖父分区局部重算 vs 宽依赖重跑上游 map stage（`Dependency.scala:52/84`）；
  重算入口 `RDD.scala:369` `computeOrReadCheckpoint`；`FetchFailedException`（`shuffle/FetchFailedException.scala`）触发 stage 重提交。
- **Optimization（横切优化 · 紫）**：仅结构化（DataFrame/SQL）走，RDD 不过。
  Catalyst：`sql/catalyst/.../analysis/Analyzer.scala:363`（`:1101` `ResolveRelations`）→
  `sql/catalyst/.../optimizer/Optimizer.scala:51`（`:100` `defaultBatches`、`:1089` `ColumnPruning`、`:2092` `PushDownPredicates`、`:248` `CostBasedJoinReorder`）→
  物理规划 `sql/core/.../execution/SparkPlanner.scala:31`（`:38` `strategies`）。
  阶段驱动 `QueryExecution.scala:211` `analyzed` → `:329` `optimizedPlan` → `:339` planning。
  Tungsten：`sql/core/.../execution/WholeStageCodegenExec.scala:643` `WholeStageCodegenExec` / `:673` `doCodeGen` /
  `:47` `trait CodegenSupport`；`:594` 注释"generates Java code on the driver side and sends it to the Executors for compilation"。
