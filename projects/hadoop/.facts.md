源码基准：**Apache Hadoop 3.6.0-SNAPSHOT**（`/Users/zhangdongdong92/workdir/hadoop/pom.xml:<version>3.6.0-SNAPSHOT`，`hadoop-project/pom.xml` 同）。

默认值（`hadoop-hdfs-project/hadoop-hdfs/src/main/resources/hdfs-default.xml`）：`dfs.replication=3`、`dfs.blocksize=134217728`(128MB)、`dfs.heartbeat.interval=3`(s)。

## 判型（system-type verdict）

**主从式（master/slave）分布式文件系统**：单一 NameNode 作元数据权威、全内存持有整棵命名空间与块映射；一群 DataNode 只存块字节；块默认三副本跨节点/跨机架容错；元数据面与数据面严格分离（客户端向 NN 要位置，再直连 DN 搬字节）。YARN 计算面叠加其上（资源调度 + 计算贴数据）。

支撑锚点：
- 主控单点、全内存：`namenode/FSNamesystem.java:389 class FSNamesystem`；持有 `:553 FSDirectory dir`、`:554 BlockManager blockManager`。
- 命名空间全内存树：`namenode/FSDirectory.java:111 class`、`:158 INodeDirectory rootDir`；文件→块列表 `namenode/INodeFile.java:66 class`、`:254 BlockInfo[] blocks`、`:665 getBlocks()`。
- 从节点只存字节：`datanode/DataNode.java:312 class DataNode`（block + `.meta` 校验和，FsDatasetImpl 多卷）。
- 三副本默认：`dfs.replication=3`；块 128MB：`dfs.blocksize=134217728`。
- 元数据/数据分离：写路径先经 NN 拿块位置 `namenode/FSNamesystem.java:2781 startFileInt`、`:3105 getAdditionalBlock`（addBlock），字节则直连 DN。

## 设计张力（explicit tension）

1. **中心化元数据·简单强一致 ⟷ 单点内存瓶颈与 SPOF**
   - 中心化：整棵 namespace + 块→DataNode 映射都在单个 NN 内存（`FSDirectory.java:158 rootDir`、`blockmanagement/BlockManager.java:340 BlocksMap blocksMap`）→ 强一致、易管、单点决策。
   - 代价：单点内存上限限制文件/块规模；NN 宕即全集群不可用（SPOF）。
   - 兜底：HA 双 NN + JournalNode 多数派 editlog。`namenode/ha/EditLogTailer.java doTailEdits`（Standby 追日志）、`namenode/ha/StandbyCheckpointer.java:63/:193 doCheckpoint`（检查点）、`qjournal/client/QuorumJournalManager.java:72`（“requiring a quorum of nodes to ack each write”）、`qjournal/server/JournalNode.java`、`tools/DFSZKFailoverController.java`（ZKFC 选主+fencing）。
2. **大块顺序吞吐（一次写多次读） ⟷ 小文件/低延迟不友好**
   - 128MB 大块 + 客户端直连 DN + 就近读 → 高顺序吞吐（`blockmanagement/DatanodeManager.java:574 sortLocatedBlocks`、`:678 sortByDistance` 排最近副本）。
   - 代价：每个文件/块的元数据都占 NN 内存，海量小文件撑爆 NN；非低延迟随机读写场景。

## 五路径归纳

- **Hot（读写数据流）**：写——client 向 NN 要块位置后，字节直连 DN，经写流水线 DN1→DN2→DN3 落三副本（`datanode/BlockReceiver.java:76 class`、`:103 mirrorOut`、`:108 downstreamDNs`、`:615 mirrorPacketTo` 逐跳转发下游；packet 流水 + ack 反向）。读——客户端读最近副本（`DatanodeManager.java:678 sortByDistance`）。数据面绕开 NN，不经主控。
- **Control（心跳 + 块调度）**：元数据 RPC（`FSNamesystem.java:2781 startFileInt`/`:3105 getAdditionalBlock` 等）；DataNode 心跳 3s 携 `DatanodeCommand`（`datanode/BPServiceActor.java:544 sendHeartBeat`、`:570 bpNamenode.sendHeartbeat`）+ 块汇报（`:386 blockReport`）；NN 侧 `blockmanagement/HeartbeatManager.java:47 class`（heartbeatCheck 判死）。
- **State（namespace + 块映射 + editlog）**：内存态——`FSDirectory rootDir`（namespace 树）、`BlockManager.java:340 blocksMap`（块↔DN）；持久态——`namenode/FSEditLog.java:127 class`（预写日志，`logSync` 先落盘再返回）+ `namenode/FSImage.java:84/:89`（“handles checkpointing and logging of the namespace edits”，FsImage 检查点）。
- **失败（副本重建 + HA 故障切换）**：`BlockManager.java:343 Daemon redundancyThread = RedundancyMonitor`，配 `neededReconstruction`/`pendingReconstruction`（`:195`）监控低/过副本并调度重建；HA 由 ZKFC 触发 Active⇄Standby 切换（`tools/DFSZKFailoverController.java` + `namenode/ha/`），fencing 防脑裂，DN 向双 NN 汇报。
- **横切（机架感知 + Balancer）**：块放置 `blockmanagement/BlockPlacementPolicyDefault.java` chooseTarget → `:557 chooseLocalStorage`(1 本地) + `:570/:579 chooseRemoteRack` + `:582/:585 chooseLocalRack`（1 本地 + 跨机架副本）；集群均衡 `balancer/Balancer.java`（Balancer 均衡 DN 利用率；Federation/RBF Router 挂多命名空间横向扩展）。
