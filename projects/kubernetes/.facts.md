## 一、判型（这是什么系统）

**声明式容器编排系统**。唯一外部接触面是**声明式 API 资源**——用户经 kubectl / client-go 只提交对象的 `spec`（期望态 desired），
从不写执行步骤；系统的职责是把 `status`（实际态 actual）持续驱动到 `spec`。整个控制面围绕一条**期望态 vs 实际态的持续 reconcile 控制循环**运转，
**etcd 是全集群唯一有状态真源**（single source of truth），**kube-apiserver 是唯一读写 etcd 的枢纽**，其余组件全部经 API Server 读写对象 + watch 变更，彼此零直接 RPC。

- 声明式而非命令式：用户说"我要 3 个副本"而非"启动第 3 个容器"。
- level-triggered（水平触发）：控制器不处理"发生了什么事件"，只对比"现在与期望差多少"，丢事件靠 resync 兜底。
- 一切扩展同构：CRD 注册新类型 + 自定义控制器复用同一 reconcile 骨架（Operator = CRD + reconcile）。

一句话判型（拟用于标题）：
**声明式容器编排系统 · 期望态 vs 实际态的持续 reconcile 控制循环 · etcd 为唯一真源、apiserver 为唯一枢纽**

## 二、设计张力（本次最大补强点 —— 必须做成显式铭牌带）

系统的根本矛盾——一句能立在图上的铭牌：

> **中心化真源 & 声明式期望态（all state in etcd，用户只声明 want）
> vs 组件解耦异步 & 最终一致的收敛延迟（controller/scheduler/kubelet 各自 watch-reconcile，
> 经 resync/乐观并发 resourceVersion 收敛，非实时强一致）**

拆解（每条都可回源码）：

1. **期望态 vs 实际态**：`spec`=用户声明的 desired；`status`=系统回报的 actual。控制器的唯一工作就是把 actual 拉向 desired。
   - reconcile 主体：`pkg/controller/deployment/deployment_controller.go:574` `syncDeployment`（读缓存当前态、对比 spec、差异回写）。
   - 节点侧同构：`pkg/kubelet/kubelet.go:2041` `SyncPod` 重算容器差异，`:2657` `syncLoop` 是节点级 reconcile。

2. **中心化真源 vs 组件解耦**：etcd 存全量声明态、apiserver 独占读写；但控制器/调度器/kubelet 互不调用，各自 watch 后独立动作。
   - 唯一枢纽的过滤器链：`staging/src/k8s.io/apiserver/pkg/server/config.go:1036` `DefaultBuildHandlerChain`。
   - 唯一后端存储：`staging/src/k8s.io/apiserver/pkg/storage/etcd3/store.go:270` `Create`、`:459` `GuaranteedUpdate`。

3. **收敛延迟 & 冲突 vs 最终一致**：异步 watch + 本地缓存必然有延迟，并发写靠**乐观并发**而非锁。
   - 乐观并发的物理实现：etcd3 用 `OptimisticPut(ctx, key, data, origState.rev, ...)`
     （`store.go:459` `GuaranteedUpdate` 把 `origState.rev` 作为期望 revision 传入，compare-and-swap；rev 变了即冲突重试）。
     revision 即对外暴露的 `resourceVersion`。
   - 冲突后重试 + 退避：`staging/src/k8s.io/client-go/util/workqueue/default_rate_limiters.go:82`
     `TypedItemExponentialFailureRateLimiter`（`baseDelay*2^failures`，封顶 `maxDelay`）；
     控制器 `deployment_controller.go:416` `AddRateLimited` 重入队，`:59` `maxRetries=15`。
   - 收敛兜底：`staging/src/k8s.io/client-go/tools/cache/shared_informer.go` resync（`resyncCheckPeriod`，:619）周期性全量重算，丢事件不致命。
