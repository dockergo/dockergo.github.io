## 一、判型（system-type verdict）

**命令式 define-by-run 张量框架 · Python 前端 + C++ Dispatcher 分发内核 + autograd 反向图。**

依据：
- 图不是预先声明再编译，而是"随 Python 代码逐算子跑出来"——每个张量算子即时执行（eager）。判型关键词 define-by-run / 命令式。
- 分发内核是灵魂：一次算子调用不直接落到某个函数，而是穿过 `Dispatcher`，按 `DispatchKeySet` 逐层 redispatch 到 kernel。
  - `aten/src/ATen/core/dispatch/Dispatcher.h:71` `class TORCH_API Dispatcher final`
  - `Dispatcher.h:179` `Return call(const TypedOperatorHandle<...>& op, Args... args)`
  - `Dispatcher.h:198` `Return redispatch(...)`（kernel 内继续下发，"treats the inputted DispatchKeySet as is, does NOT mask"）
  - `Dispatcher.h:204` `void callBoxed(const OperatorHandle& op, Stack* stack)`
- 反向是"图"：前向留 `grad_fn` 节点，反向由引擎逆序遍历 Node/Edge 求梯度。
  - `torch/csrc/autograd/engine.h:130` `struct TORCH_API Engine`
  - `torch/csrc/autograd/function.h` `struct Node` / `edge_list next_edges`（`function.h:20`）

## 二、设计张力（design tension）

**动态图易用（eager 逐算子下沉 · Python 随代码建图 · 可断点调试）  ⚖  静态图性能（torch.compile 捕获 FX 图 → Inductor 融合内核）。**

依据：
- 动态图代价：每算子一次 Python→C++→Dispatcher 往返，逐算子分发有开销、无跨算子融合。
- 性能补偿走横切编译栈，而非改主执行模型：
  - `torch/_dynamo/eval_frame.py:63` `set_eval_frame`（挂 CPython 帧求值钩子，抓字节码）
  - `torch/_dynamo/convert_frame.py:135` `GuardedCode`（带 guard 的编译产物；guard 失效则重编译，抓不动则 graph break 回退 eager）
  - `torch/_inductor/compile_fx.py:800` `def compile_fx_inner`，`compile_fx.py:125` `from .graph import GraphLowering`（FX 图 → 融合 kernel）
  - `torch/_inductor/scheduler.py`（融合调度）
- 张力落点：默认 eager 保灵活/可调试；需要性能时用 `torch.compile` 换静态捕获与融合，二者共存而非二选一。

## 三、五路径归纳

### 1) Hot 前向（逐算子下沉，绿）
用户张量算子 → Python 前端 → C++ Dispatcher → ATen kernel → 设备。
- 入口态：`c10/core/TensorImpl.h:510` `struct C10_API TensorImpl`（shape/stride/dtype 元数据 + 指向 Storage）
- 算子 schema：`aten/src/ATen/native/native_functions.yaml`（~算子定义与注册；文件确认存在，630KB）
- 落地计算：`aten/src/ATen/native/`（CPU 向量化 / structured kernel）

### 2) Control 分发（Dispatcher 逐层 redispatch，蓝）
按 `DispatchKeySet` 从高优先级功能键逐层下发到最终 backend kernel。
- `c10/core/DispatchKey.h:136` `enum class DispatchKey : uint16_t`
  - `DispatchKey.h:222` `BackendSelect`
  - `DispatchKey.h:443` `Autograd`（alias key），运行时如 `AutogradCPU`
  - Autocast 混合精度亦为一层功能键
- `c10/core/DispatchKeySet.h:167` `class DispatchKeySet final`
- `aten/src/ATen/core/dispatch/DispatchKeyExtractor.h:24` `computeDispatchKeySet(...)`（从算子入参算出 keyset）
- 每算子一张 kernel 表：`aten/src/ATen/core/dispatch/OperatorEntry.h:70` `class TORCH_API OperatorEntry final`；`OperatorEntry.h:237` `std::array<KernelFunction, c10::num_runtime_entries> dispatchTable_`

### 3) State 设备/内存（张量状态与显存池化，琥珀）
- 存储：`c10/core/StorageImpl.h:55` `struct C10_API StorageImpl`（底层字节，intrusive_ptr）；TensorImpl view 共享同一 Storage
- 显存池化：`c10/cuda/CUDACachingAllocator.cpp`
  - 块与流绑定、可复用、可 split；`>= max_split_size` 的大块不再 split 以降碎片（`CUDACachingAllocator.cpp:86-101` 注释）；无合适块则 delegate 到 `cudaMalloc`
  - 配置：`c10/cuda/CUDAAllocatorConfig.cpp`

### 4) 反向 autograd（引擎逆序回灌梯度，橙）
- `torch/csrc/autograd/engine.h:130` `struct TORCH_API Engine`；`engine.h:152` `virtual variable_list execute(...)`
- `engine.h:26` `struct ReadyQueue`、`GraphTask`（多线程按依赖就绪调度）
- 反向节点：`torch/csrc/autograd/function.h` `struct Node`、`Edge`、`next_edges`（`function.h:20` `edge_list next_edges`）
- 实现：`torch/csrc/autograd/engine.cpp`
- 触发面：`torch/autograd/__init__.py`（`backward()` / `grad()`）；张量侧 `c10/core/TensorImpl.h:1990` `set_autograd_meta(...)`

### 5) 编译横切（torch.compile，紫，旁路非主栈）
- Dynamo 抓帧：`torch/_dynamo/eval_frame.py`、`convert_frame.py`、`guards.py`、`output_graph.py`
- Inductor 降级融合：`torch/_inductor/compile_fx.py`、`graph.py`（`GraphLowering`）、`scheduler.py`、`lowering.py`
- 抓不动则 graph break 回退 eager；guard 失效则重编译

### 横切其余（旁路）
- 分布式：`torch/distributed/distributed_c10d.py`（ProcessGroup / collective）、`torch/distributed/fsdp/`、`torch/distributed/_composable/`（DDP/FSDP）
- 数据加载：`torch/utils/data/dataloader.py`、`dataset.py`（多 worker 预取 · pin_memory）

## 四、接口层锚点
- 张量编程：`c10/core/TensorImpl.h`（Python 面 `torch`）
- 建模与训练：`torch/nn/modules/module.py`（nn.Module）、`torch/optim/`（optimizer）
- 自动微分：`torch/autograd/__init__.py`（requires_grad / backward）
