## 判型（题眼）
- **控制面 / 数据面分离**：JobManager 只调度协调、不碰数据；TaskManager 跑算子 + shuffle，slot 为资源单位。
  - 控制面：`flink-runtime/.../jobmaster/JobMaster.java`、`.../resourcemanager/ResourceManager.java`、`.../checkpoint/CheckpointCoordinator.java`。
  - 数据面：`flink-streaming-java/.../streaming/runtime/tasks/StreamTask.java`（算子执行）、`flink-runtime/.../io/network/partition/ResultPartition.java`（shuffle）。
- **编译脊**：StreamGraph → JobGraph → ExecutionGraph（逻辑 DAG → 链化 → 并行展开），两 API（DataStream / Table-SQL）汇入同一物理管线。

## 设计张力（对立面命名牌）
- **精确一次容错（exactly-once）** —— 分布式快照 / CheckpointBarrier 随流注入 / 状态一致可恢复
  - vs
- **低延迟·高吞吐** —— 异步快照不停流 / credit 背压不阻 TCP / mailbox 单线程免锁
- 协调点：屏障对齐（可切非对齐 unaligned）· 同步建资源 + 异步写快照 · RocksDB 增量检查点。
- 源码佐证：
  - `SubtaskCheckpointCoordinatorImpl.java:344` 构造 `CheckpointBarrier` → `operatorChain.broadcastEvent(checkpointBarrier, options.isUnalignedCheckpoint())`（barrier 随流注入 + 对齐/非对齐）。
  - 同文件 Step(5) 注释：“Take the state snapshot. This should be largely asynchronous, to not impact progress of the streaming topology”（异步快照不停流）。
  - `StreamTask.java:296` `asyncOperationsThreadPool`，`:1305` `triggerCheckpointAsync(...)`（快照异步落盘）。
  - `MailboxProcessor.java:48` 注释：“single-threaded execution”，核心循环 `runMailboxLoop()`（记录/检查点/定时器串行、免加锁）。
  - `BufferManager.java:172` credit 语义（announce credit to producer）→ 背压靠 credit，不阻塞 TCP。

## 五路径归纳
1. **Hot 记录流**：外部 Source → 算子链（source→map，forward 链化）→ 跨子任务 shuffle（ResultPartition→InputGate）→ Sink。
   - `ResultPartition.java`、`consumer/SingleInputGate.java`、算子链执行 `StreamTask.java`。
2. **Control 调度 + 反压**：API → 三级图变换 → JobMaster 提交 ExecutionGraph → RM/SlotManager 给 slot → 部署 task；反压走 credit（BufferManager）。
   - `jobmaster/JobMaster.java`、`resourcemanager/ResourceManager.java` + `resourcemanager/slotmanager/SlotManager.java`、`consumer/BufferManager.java`。
3. **State 检查点快照 + 状态后端**：CheckpointCoordinator 定时触发 → barrier 注入各算子 → 异步 snapshot → 状态后端持久化。
   - `checkpoint/CheckpointCoordinator.java:568 triggerCheckpoint(...)`、`io/network/api/CheckpointBarrier.java`、`SubtaskCheckpointCoordinatorImpl#broadcastEvent`。
   - 后端：`state/hashmap/HashMapStateBackend.java`（堆内小状态低延迟）/ `flink-statebackend-rocksdb/.../EmbeddedRocksDBStateBackend.java`（盘 + 增量检查点，`:137 incremental`）。
   - 分片：`state/KeyGroupRangeAssignment.java:50 assignKeyToParallelOperator(...)`（key-group，rescale 基础）；keyBy 断链 `streaming/runtime/partitioner/KeyGroupStreamPartitioner.java:63`。
4. **失败恢复 / 重分布**：失败后 CheckpointCoordinator 从最近 checkpoint 重部署，状态后端按 key-group 交集从 DFS 回填。
   - CheckpointCoordinator 驱动恢复（`restoreLatestCheckpointedState` 语义）；状态从检查点存储（HDFS/S3）回填到 state backend。
5. **横切 时间与窗口**：watermark / window（assigner+trigger+状态），落在数据面算子链内（tid=time，window→sink 节点）。
