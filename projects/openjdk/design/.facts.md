## 判型
**托管运行时虚拟机（HotSpot JVM）**：类加载（双亲委派 + 解析校验）＋ 分层执行（解释器 → C1 → C2 JIT，按热度自适应升级）＋ 可插拔并发 GC（CollectedHeap/BarrierSet 抽象，G1/ZGC/Shenandoah/Parallel）＋ safepoint/handshake 全局协调 —— 一次编译到处运行的 JVM。

## 设计张力
**托管抽象**（GC 自动内存 + JIT 自适应优化 + 平台无关字节码，一次编译到处运行）
　⟷ vs ⟷　
**停顿 / 预热 / 内存开销**（GC pause、JIT warmup、safepoint STW、对象头+元数据内存税）——靠分层编译（启动用解释器/C1、峰值用 C2）+ 并发 GC（标记/疏散与应用并跑）缓解。

## 五路径归纳（总架构图例语义）
- **Hot 绿（执行流 解释→JIT）**：字节码解释器执行 → 热点方法/循环计数器超阈 → C1 → C2 逐级升级编译成机器码。
- **Control 蓝（类加载 + 编译派发）**：双亲委派加载 Klass 装入执行引擎；CompileBroker 编译队列按计数器派发异步编译。
- **State 琥珀（堆 + 对象模型）**：执行引擎在 Java 堆按 oop-klass 布局分配/读写对象；对象经 BarrierSet 屏障访问。
- **失败 橙（deopt + GC-pause）**：C2 激进优化假设失效 → uncommon_trap 去优化回退解释器；GC 回收伴随 STW 停顿。
- **横切 紫（safepoint + JMM）**：safepoint/handshake 暂停 Java 线程作 GC/deopt 前置；JMM happens-before + 内存屏障约束可见性与重排序。
