## 一、判型（这套系统"是什么型"）

**数据流图执行引擎**：TF 是一个"张量数据流图 + 自动微分"框架。计算被建模为有向图，节点=算子(op)、边=张量(tensor)；
运行时以**数据流就绪调度**驱动——某节点所有输入边就绪(入度归零)即可发射。

- 默认执行档＝**eager 即时**：Python 每调一个 op 就经 `python/eager/execute.py:quick_execute` → `pywrap_tfe.TFE_Py_Execute` 直发到 C++ 运行时，逐算子返回，无需先建图。
- 加速执行档＝**graph 编译**：`@tf.function` 把 Python 函数**追踪(trace)成 FuncGraph**，整图交给编译管线(Grappler 重写 / 可选 XLA 融合 / Placer 定设备)，产出可复用的优化图，再交 `Executor` 数据流调度。

判型一句话：**默认 eager 逐算子即时发射；`@tf.function` 追踪成 FuncGraph 编译优化；Executor 按入度归零就绪调度。**

## 二、设计张力（贯穿全图的第一矛盾）

**eager 即时执行 ⇄ graph 整图编译** —— TF 2.x 的定义性二元对立：

| 维度 | eager（默认） | graph（`@tf.function`） |
|---|---|---|
| 触发 | 每 op 直发 | 函数首次调用追踪成图 |
| 易用 | 即时求值、Python 原生调试、断点可看中间张量 | 图是符号的，调试隔一层 |
| 性能 | 无跨 op 优化，Python 开销/op | 整图重写(Grappler)、算子融合(XLA)、常量折叠 → 高吞吐 |
| 部署 | 依赖 Python | 图可序列化 SavedModel、脱 Python 部署 |
| 代价 | 慢 | 追踪成本 + 变签名重追踪(retrace) |

一句话张力：**eager 即时易调试灵活 ⇄ graph 整图编译(Grappler 重写 / XLA 融合)高吞吐可部署。**
其余张力：不可变 Tensor ⇄ 有状态 Variable；同步就绪调度 ⇄ 异步发射；单机 ⇄ 分布式镜像。

## 三、五路径归纳（节点按语义着色）

### 路径 A — Hot 主执行流（graph，绿）：一次 graph 计算真正跑起来
1. `@tf.function` 函数被调用 → 追踪 → FuncGraph
   - `python/eager/polymorphic_function/polymorphic_function.py:Function`（`class Function(core.PolymorphicFunction, trackable.Trackable)`，L453；`__call__` L809）
   - `python/framework/func_graph.py:func_graph_from_py_func`（L921）；`class FuncGraph(ops.Graph)`（L134）
2. 编译后图交执行引擎，**入度归零即就绪**，发射节点、传播输出
   - `core/common_runtime/executor.cc:ExecutorImpl`（L148）、`ExecutorState`（L283）
   - `core/common_runtime/pending_counts.h:PendingCounts`（记 pending 数，`decrement_pending`→0 即就绪，L29 起）
3. 每个就绪节点查 kernel、`Compute()` 落设备
   - `core/framework/op_kernel.cc:FindKernelRegistration`（L1447）
   - `core/framework/op_kernel.h:OpKernel::Compute`（纯虚 `virtual void Compute(OpKernelContext*) = 0`，L158）

### 路径 B — Control 图编译（蓝）：整图拿到手后离线重写/编译/定位
1. Grappler MetaOptimizer 串十余 pass 整图重写
   - `core/grappler/optimizers/meta_optimizer.cc:MetaOptimizer`（`class MetaOptimizer : public GraphOptimizer`，.h L35）
   - pass：`constant_folding.cc` / `arithmetic_optimizer.cc` / `generic_layout_optimizer.cc` / `remapper.cc`
2. XLA/JIT（可选）自动聚类子图 → 编译成融合大 kernel
   - `compiler/jit/mark_for_compilation_pass.cc:MarkForCompilationPass`（.h L36）
   - `compiler/jit/encapsulate_subgraphs_pass.cc`；插入 `_XlaCompile` / `_XlaRun`
3. Placer 按 colocation 组 + soft placement 定 CPU/GPU/TPU
   - `core/common_runtime/placer.cc:Placer`（.h L57）

### 路径 C — State 张量与内存（琥珀）：数据/显存怎么存与复用
- Tensor = 形状/类型 + 共享引用计数缓冲
  - `core/framework/tensor.h:Tensor`（L45）；`TensorBuffer : public core::RefCounted`（L72）
- BFC 显存池（best-fit with coalescing）
  - `core/common_runtime/bfc_allocator.h`（TF 侧转发头，`#include "xla/tsl/framework/bfc_allocator.h"`）
  - 类实现在 `third_party/xla/xla/tsl/framework/bfc_allocator.cc:BFCAllocator`
- op 注册 schema：`core/framework/op.h:REGISTER_OP`（L318）；kernel 注册：`core/framework/op_kernel.h:REGISTER_KERNEL_BUILDER`（L1469 起 IMPL）

### 路径 D — eager 旁路（琥珀虚线）：跳过整个编译管线
- `python/eager/execute.py:quick_execute`（L28）→ `pywrap_tfe.TFE_Py_Execute`（L53）逐算子直发；`python/eager/context.py` 持 eager 上下文

### 路径 E — 横切（紫）：autodiff + 分布式，缠绕主流两档
- 自动微分 GradientTape：`python/eager/backprop.py:GradientTape`（L705）；`imperative_grad.imperative_grad`（逆序回放录制求梯度，L1066）
- 分布式 tf.distribute：`python/distribute/distribute_lib.py:StrategyBase`（`scope` L1223 / `run` L1557）；`python/distribute/mirrored_strategy.py`（镜像变量）；`python/distribute/collective_all_reduce_strategy.py`（all-reduce 聚梯度）
