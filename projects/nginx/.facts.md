# nginx 源码核实事实台账（source-verified fact ledger）

> 源码：`~/workdir/nginx`(src/ 下 core/ event/ http/ os/unix/)。每条事实从真实源码 grep/read 核实,标 `file:line`。
> 判型:**事件驱动 master-worker 反向代理/Web 服务器**(event-driven reverse proxy) —— 一个 master 管控 + N 个单线程 worker,每 worker 一条 epoll/kqueue 事件循环非阻塞地多路复用海量连接(C10K)。
> 设计张力:**单线程 worker 无阻塞高并发 vs 任何阻塞调用都会卡死整条 worker** —— 全异步回调 + 内存池 + 零拷贝换吞吐,代价是编程复杂度与"绝不阻塞"的纪律。
> 次级张力:**master-worker 隔离/热重载 vs 共享状态协调**(accept_mutex、shared-memory zone);**sendfile 零拷贝高吞吐 vs 灵活性**。

## 主线 12 · 优雅重启/信号(graceful reload · binary upgrade · Failure/Control)

- master 循环消费标志(`ngx_process_cycle.c`):

## 五路径归纳(供绘图 · 总架构图重画依据)
- **Hot Path**:client → worker epoll_wait(event.c:195/epoll:800)→ ngx_event_accept → HTTP 解析(process_request_line/headers)→ 11 阶段 run_phases(core_module.c:884)→ content handler → body filter 链 → write_filter/sendfile;反代则经 upstream_connect→process_header→send_response。
- **Control Path**:master 信号(HUP 重载 / USR2 换二进制)、accept_mutex 抢锁分连接、config 解析装配指令、upstream_next 选后端。
- **State Path**:每请求内存池(palloc/destroy)、buffer chain、共享内存 zone(accept_mutex / proxy_cache rbtree / limit_req rbtree)、定时器红黑树。
- **Failure Path**:worker 崩溃→master reap 重生;后端失败→upstream_next 切换;超时→timer rbtree 触发 handler;过载→accept_disabled 让锁 + limit_req 拒绝;阻塞调用→卡死整 worker(最大风险,靠 thread_pool/AIO 外移)。
- **Optimization**:epoll 边缘触发(EPOLLET)、accept_mutex 防惊群 + 连接均摊、NGX_POST_EVENTS 延后减锁期、内存池碰撞分配、sendfile 零拷贝、smooth weighted RR、pipelined filter chain。

## 总架构图重画蓝图(五路径工业剖面)
顶层 clients ──(共享 listen sockets)──> 一排 N 个 **worker 进程**(每个内含一台"事件循环引擎"),上方独立 **master** 只连信号/管控线(HUP/USR2/reap),不碰数据面。
放大任一 worker 的工业剖面(中央 = epoll 事件循环心脏 `ngx_process_events_and_timers`,四周挂):
- 左 **Control 栏**:accept_mutex(共享内存,防惊群/均摊)、config 指令表、master 信号入口。
- 中 **Hot 主轨**(横向流水线):accept → HTTP 解析 → **11 阶段相位引擎**(POST_READ…CONTENT…LOG,模块挂 handler)→ content handler → **header/body filter 双链**(gzip/chunked/range→write_filter)→ sendfile 零拷贝出 socket;下挂 **upstream 支线**(connect→send→process_header→send_response,平滑加权 RR + upstream_next 故障切换)。
- 右 **State 栏**:每请求内存池(create/palloc/destroy 生命周期)、buffer chain、定时器红黑树、共享内存 zone(proxy_cache rbtree + limit_req leaky-bucket)。
- 底 **Failure 栏**:worker 崩溃→master 重生;后端故障→切 peer;超时→timer 触发;"任一阻塞调用卡死整 worker"红色警示 → thread_pool/AIO 外移。
- 全图核心张力标注:**单线程 worker 全异步非阻塞换 C10K 高并发,纪律是绝不阻塞;master-worker 隔离换零停机热重载,共享状态靠 shmem+锁协调**。
