# Apache Arrow 核心原理 · 源码核实 facts

**版本基准**：`cpp/CMakeLists.txt` → `set(ARROW_VERSION "26.0.0-SNAPSHOT")`；`cpp/src/arrow/util/config.h.cmake` 定义 `ARROW_VERSION_MAJOR/MINOR/PATCH`。核实基准 v26.0.0-SNAPSHOT（本地 clone）。

## 判型
**语言无关的列式内存格式 + 零拷贝数据交换标准**：列式 Array(validity bitmap + offset + value buffers) + IPC / C Data Interface 跨进程跨语言零拷贝 + 向量化 compute kernel / Acero 流式执行。

## 设计张力（显式铭牌）
**标准化列式内存布局 + C Data Interface（免序列化、跨语言直接共享内存）换来生态零拷贝互通　⟷ vs ⟷　列式布局对随机行访问 / 逐行写入 / 频繁更新不友好；与行式系统互转有转换成本。**

## 五路径归纳（arch legend）
- Hot 绿：列式数据 / 计算流（Array buffers → kernel → Array）
- Control 蓝：schema + 执行计划（DataType 描述 Array；Acero ExecPlan 编排）
- State 琥珀：buffers + 内存池（Buffer 由 MemoryPool 64B 对齐分配、引用计数持有）
- 失败 橙：空值 / 校验（validity bitmap 空值语义、ValidateFull）
- 横切 紫：IPC + C-Data + Flight 零拷贝交换
