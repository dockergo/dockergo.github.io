## 一、判型（是什么）

**工业级容器运行时守护进程**：一个 `containerd` 长驻进程，内部所有能力（内容存储/快照/差分/运行时/CRI…）都是**插件**，经**依赖图拓扑排序**依次初始化、用共享 plugin set 做依赖注入；对外暴露 **gRPC/TTRPC 服务**；容器的真正拉起交给**每容器一个独立 shim 进程**（runtime v2），shim 再调 runc/OCI。被 kubernetes 经 **CRI plugin** 驱动。

关键锚点：
- 插件类型全表 `plugins/types.go`：`ContentPlugin="io.containerd.content.v1"`、`SnapshotPlugin="io.containerd.snapshotter.v1"`、`RuntimePluginV2="io.containerd.runtime.v2"`、`GRPCPlugin="io.containerd.grpc.v1"`、`CRIServicePlugin="io.containerd.cri.v1"`、`MetadataPlugin`、`DiffPlugin`、`ShimPlugin`、`LeasePlugin` 等（每个子系统一种 plugin.Type）。
- 守护进程装配 `cmd/containerd/server/server.go:124 New` → `:135 LoadPlugins` → `:161` 遍历 loaded → `:166 plugin.NewContext`（注入 root/state/grpc 地址）→ `:189 result := p.Init(initContext)` → `:190 initialized.Add(result)` → `:194 result.Instance()`。
- 拓扑序来源 `cmd/containerd/server/server.go:313 LoadPlugins` → `:386 registry.Graph(filter(config.DisabledPlugins))`。

## 二、设计张力（矛盾/取舍）

**插件化解耦·可扩展**（每子系统一 plugin、shim 隔离容器生命周期）　⟷ vs ⟷　**守护进程重启的状态一致**（shim 是独立进程，容器不随 containerd 重启而死；重启后按落盘 bootstrap.json 重连回来）。

关键锚点：
- 解耦：`vendor/github.com/containerd/plugin/plugin.go:68 Requires []Type`（插件只声明依赖类型，不硬编码具体实现）；`registry.Graph` 按 Requires 排序，`children`（`plugin.go:137`）递归先装依赖。
- shim 独立进程：`core/runtime/v2/binary.go:66 Start` 启动 shim 二进制，`:117 cmd.CombinedOutput()` 拿到 shim 自报的 socket；`:142 writeBootstrapParams(... "bootstrap.json")` 落盘连接参数；`:127` 落盘 `shim-binary-path`。
- 重启存活/重连：`core/runtime/v2/shim_load.go:38 LoadExistingShims` → `:65 loadShims` → `:127 loadShim`（daemon 重启后从 state 目录扫已存在 shim 重新接管）；`:172 cleanupAfterDeadShim`（仅清理真已死的 shim）。

## 三、五路径归纳（图例语义）

1. **Hot 拉起/运行流（绿）**：client `Pull`→content 落盘→`Unpack`→snapshot；`task Create`→shim `Start`→runc 起容器。锚：`client/pull.go:43 Pull`、`:134 unpack.NewUnpacker`；`plugins/services/tasks/local.go:171 Create`→`:268 l.v2Runtime`；`core/runtime/v2/task_manager.go:159 Create`→`:213 m.manager.Start`；`cmd/containerd-shim-runc-v2/task/service.go:223 Create`、`:296 Start`。
2. **Control 蓝（gRPC + 插件分发）**：`server.go:189 p.Init` 依赖注入；gRPC 服务 `plugins/services/tasks/local.go`；CRI 分发 `plugins/cri/cri.go:48 registry.Register{Type:GRPCPlugin,ID:"cri"}`（`:51 Requires` 列 11 种 plugin.Type，DI 拿齐依赖）。
3. **State 琥珀（内容寻址 + 快照 + 元数据真源）**：`plugins/content/local/store.go:646 blobPath` → `blobs/<algo>/<digest>`（内容寻址，摘要即路径）；`core/snapshots/snapshotter.go:59 KindView/:60 KindActive/:61 KindCommitted`（CoW 生命周期）；`core/metadata/db.go:120 NewDB`（bbolt 事务库统管 content/snapshot 引用）。
4. **失败 橙（shim 存活于 daemon 重启）**：`shim_load.go:38 LoadExistingShims`（重启重连）；`binary.go:142 bootstrap.json`（落盘连接参数）；`shim_load.go:172 cleanupAfterDeadShim`（只清真死的）。
5. **横切 紫（CRI + 事件总线）**：`internal/cri/server/sandbox_run.go:54 RunPodSandbox`、`container_create.go:60 CreateContainer`（k8s 接触面）；`core/events/exchange/exchange.go:80 Publish`、`:128 Subscribe`、`:55 Forward`（全局事件扇出）。

## 四、重画蓝图（分层脊）

- ① 接触面：gRPC/TTRPC API + `ctr` 客户端 + **CRI plugin**（k8s 经此驱动）。锚 `internal/cri/server/service.go:68 CRIService`、`:120 criService`。
- ② 守护进程 + 插件图：`server.go:124 New`/`LoadPlugins`/`registry.Graph`；plugin 类型全表 `plugins/types.go`。
- ③ 核心服务（元数据侧，只读写对象）：containers `core/containers/containers.go:30 Container`（:44 Image/:50 Runtime/:64 SnapshotKey）、images、content、snapshots、diff、leases `core/leases/lease.go:32 Manager`/`:36 AddResource`。

- ⑤ 运行时下沉：task/exec `plugins/services/tasks/local.go` → runtime v2 `task_manager.go`/`shim_manager.go:208 Start` → shim 进程 → runc/OCI。
