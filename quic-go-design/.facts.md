## 五路径归纳(供绘图)
- **Hot Path**:UDP 收包 → 解复用(transport)→ 解密解包 → handleFrame 分发 → 流重组/Read;Write → framer → composeNextPacket → AEAD → sendQueue → UDP。
- **Control Path**:握手事件队列、scheduleSending、ACK 调度、PATH_CHALLENGE/RESPONSE、MAX_STREAMS/MAX_DATA。
- **State Path**:三 PN 空间 + 加密级 key + 流 offset/gap + FC 窗 + 拥塞 cwnd/ssthresh。
- **Failure Path**:丢包检测(时间/重排阈值)→ 重传;PTO 探测;不可解密→缓冲;连接错误→destroyImpl;迁移→重置拥塞。
- **Optimization**:单 goroutine 无锁 actor、包合并 + GSO、延迟 ACK、pacing、HyStart、窗自动调、0-RTT、MTU 发现、懒建 pathManager。
