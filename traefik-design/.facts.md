# Traefik 源码核实事实台账（source-verified fact ledger）

> 源码：`~/workdir/traefik`(traefik/traefik，v3.7.7 / `git describe`=v3.7.7-32-g901642835，codename cheddar，Go 1.26；核心在 `pkg/`)。每条事实从真实源码 grep/read 核实，标 `file:line`。
> 判型:**动态自动发现的云原生边缘路由器/反向代理**(dynamic auto-configuring edge router) —— Provider 从 Docker/K8s/文件/KV 实时**发现**路由配置，推入 channel，ConfigurationWatcher 去重节流后**原子热重建**整张路由表，**零重启**。
> 设计张力:**动态自动发现 + 零停机热加载(provider watch → 实时重建路由表，无需重启) vs 一致性/抖动风险(配置频繁变更需 debounce 节流 + 原子切换避免半更新)** —— 与 nginx「静态配置 + reload 信号」正相反,Traefik 差异化在于活的 provider→watcher→router 重建闭环。
> 次级张力:**声明式动态配置(贴合容器/K8s 弹性编排) vs 静态配置的可预测性**;**中间件链组合灵活性 vs 每请求链路开销**。

## 五路径归纳(供绘图)
- **Hot Path**:client → `TCPEntryPoint.Accept`(server_entrypoint_tcp.go:240)→ 每连接 goroutine → `switcher.ServeTCP`(:287,peek 判 TLS)→ `HTTPHandlerSwitcher.ServeHTTP`(原子取当前 handler)→ `Muxer.ServeHTTP`(mux.go:47,按 priority 遍历 `matchersTree.match`)→ recovery/observability/中间件链(alice)→ `Service.BuildHTTP` 的 LB(wrr `nextServer` EDF 堆)→ proxy → 后端。
- **Control Path**:**provider 发现**(Docker/K8s/File `Provide`→channel)→ 聚合器节流(`maybeThrottledProvide` Sleep)→ `ConfigurationWatcher`(receive 去重 → apply 合并)→ listeners:`switchRouter`→`CreateRouters`(cancel 旧 + 建新)→ `TCPEntryPoints.Switch`→`HandlerSwitcher.UpdateHandler` 原子换;信号优雅停(server.go)。
- **State Path**:两张配置面(static 只读 / dynamic 热态 `Configurations` map);`safe.Safe`(RWMutex)持当前路由 handler;TLS 证书库 `CertificateStore`(SNI 缓存);LB 服务器池 + 健康态;全程 `DeepCopy` 快照。
- **Failure Path**:后端失败→retry/failover 切换 + 健康检查摘除;provider 抖动→节流+去重吸收;配置错→router 标 Disabled/Warning(`ParseRouterTree` 环检测/可达性)不影响全局;panic→`recovery` 中间件兜底;热重载失败→保留 `lastConfigurations`(不半更新);证书签发失败→ThrottleDuration 限频重试。
- **Optimization**:两级 DeepEqual 去重 + Sleep 节流(避免无谓重建)、`cancelPrevState` 取消上一代构建、`routerHandlers`/`GetBestCertificate` 缓存、WRR EDF 平滑加权、非阻塞 select 送最新态、per-provider ThrottleDuration、FastProxy(experimental)。

## 总架构图重画蓝图(五路径工业剖面)
顶层左侧一排 **发现源/Provider**(Docker、Kubernetes、File、Consul/Nomad/KV…)——各自 `Provide` 把「后端拓扑」翻译成 `dynamic.Message` 推入**同一根 channel**;这是 Traefik 区别 nginx 的命门,画成「多源 → 汇流管道」。

中枢画 **ConfigurationWatcher 引擎**(全图心脏,横向流水线):
- 入口 **聚合器节流阀**:`maybeThrottledProvide`(Sleep 限频,红色「防抖动」标注)。
- `receiveConfigurations`(**去重①**:每 provider `DeepEqual` 只留最新)→ transformers → `applyConfigurations`(等 `internal` required、**去重②** 全局 DeepEqual、`mergeConfiguration` 合并多源)→ 广播给一排 **listeners**(TLS 装配 / 指标 / switchRouter / ACME…)。
- 关键动作 **原子热切换**:`switchRouter`→`CreateRouters`(先 `cancelPrevState` 取消旧世代,再全量重建)→`TCPEntryPoints.Switch`→`HandlerSwitcher.UpdateHandler`(`safe.Safe` RWMutex 一次性换指针)——**零停机、无半更新**(粗箭头 + 「原子」印章)。

底部数据面 **一排 EntryPoint**(:80/:443… `Accept` 循环)→ 经 `HandlerSwitcher`(热点:指向「当前世代路由表」的可原子替换指针)→ 放大主轨:
- **Router/Muxer**(按 priority 排序的规则树 `Host&&Path…`,最长最具体优先)→ **中间件链**(alice 洋葱:recovery/auth/ratelimit/retry/stripprefix/headers…)→ **Service LB**(WRR-EDF/P2C/HRW + 主动/被动健康检查)→ proxy → 后端 pods/containers。
- 右侧 **TLS/ACME 支线**:SNI→`CertificateStore.GetBestCertificate`;域名→ACME 自动签发→证书回灌同一 channel(证书也是「被热加载的动态配置」)。
- 贯穿 **可观测性切面**:entrypoint/router/service 三层 metrics + tracing span + accesslog,以中间件形式内嵌链中。

全图核心张力标注:**动态自动发现 + 零停机热加载(provider watch → DeepCopy 快照 → 节流去重 → 原子重建整张路由表,全程无重启)vs 抖动/一致性风险(靠两级 DeepEqual 去重 + ThrottleDuration 限频 + safe.Safe 原子切换化解)**;对照牌:nginx = 静态配置 + SIGHUP 人工 reload;Traefik = 活的 provider→watcher→router 自动闭环。
